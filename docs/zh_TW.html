解決 AI 多帳號的救星

Orrery 把你的 Claude Code、Codex、Gemini CLI 帳號集中成一個池，一行指令就能切換 — 不用登出、不用重新認證，對話從你離開的地方接續。

開始使用 GitHub
原生安裝（macOS、Linux、WSL）— 推薦
$ curl -fsSL https://offskylab.github.io/Orrery/install.sh | bash
或透過 Homebrew 安裝（macOS）
$ brew install OffskyLab/orrery/orrery
Windows — 透過 WSL
Windows 上的 Claude Code 跑在 WSL 裡。請先以系統管理員身份開啟 PowerShell 啟用 WSL，再到 WSL shell 裡用上面的原生安裝指令安裝 Orrery。
PS> wsl --install
啟用 Shell 整合
$ source ~/.orrery/activate.sh
從 APT 遷移（Linux，v2.3.x 或更早）

如果你之前用 APT（apt install orrery）安裝，跑 orrery update 可能會看到 already the newest version (2.3.x)。APT repo 已不再維護，而且舊版的 update 流程沒有先跑 apt update。只要跑一次原生安裝指令就能完成遷移：

$ curl -fsSL https://offskylab.github.io/Orrery/install.sh | bash

這會移除 APT 管理的舊 binary、安裝新的 orrery-bin，並把之後的 orrery update 切到原生安裝流程。你可以順手清掉已失效的 APT 設定：

$ sudo rm /etc/apt/sources.list.d/orrery.list
$ sudo apt update

為切換 AI 帳號而生

🗂️

帳號池

所有登入集中在一處。Claude、Codex、Gemini 的帳號各加一次，Orrery 把憑證存進共享的池子，隨時可切。

⚡

一行指令切換

orrery use --claude work 換掉某工具當前的帳號。不用登出、不用密碼、不用瀏覽器認證流程 — 瞬間完成。

💬

Session 跟著你

做到一半切換帳號，claude --resume 照樣能用。對話歷史預設在切換之間共享。

👻

對話中無痛切換

用 orrery run claude 啟動，在 Claude 裡用 /orrery:phantom 直接切 account 或 sandbox，不會中斷對話。

📦

Sandbox 做隔離

需要每個客戶或專案有完全獨立的 config？optional 的 sandbox 在帳號之上再隔離 memory、session 與環境變數。

🐚

Per-shell，不跑常駐

切換只影響當前 shell。其他終端機視窗維持各自的帳號，互不影響 — 而且沒有任何背景程式。

🚀

零執行開銷

Swift 原生編譯，啟動瞬間完成。Shell 整合只是一個輕量的 eval 函式。支援 bash 與 zsh。

帳號池

每個 AI 工具的帳號加一次就好。Orrery 把憑證存進共享的池子 — 切換是瞬間的，不用登出、不用重新認證。

$ orrery add --claude
# 登入一次 — 帳號就加進池子

$ orrery list
claude accounts:
  - work     you@company.com, team
  - personal you@gmail.com, pro

$ orrery use --claude personal
# Claude 當前帳號 → personal，瞬間切換

核心模型

從帳號開始 — 大多數人需要的就只有這個。 只有當某個情境需要完全獨立的 config 時，才會用到 sandbox。

👤

Account（帳號）

某個工具的身分 — Orrery 用來登入的憑證。帳號住在共享的池子裡：加一次，之後隨時用 orrery use 切換。這是你每天會碰的那一層。

📦

Sandbox（沙盒）進階

optional 的隔離層：獨立的 memory、session 與環境變數。多數使用者用不到 — 當某個客戶或專案需要自己的 config 空間時才需要。用 orrery enter 進入、orrery exit 離開。

💬

Session

代表延續性：對話紀錄與專案 context。預設在帳號切換之間共享 — claude --resume 直接就能用。

傳統工具隔離的是相依性（virtualenv）或 runtime 版本（nvm）。 Orrery 把這個概念延伸到 AI 帳號 — 需要時，再加上完整的 config 隔離。

運作方式

1

安裝 Shell 整合

執行一次 orrery setup。會產生 ~/.orrery/activate.sh 並寫入 rc 檔。再執行 source ~/.orrery/activate.sh 即可在當前 shell 生效。

activate.sh 也會在背景每天檢查是否有新版本，並在下次開啟 shell 時顯示通知。

2

加入你的帳號

執行 orrery add --claude（或 --codex / --gemini）並登入一次，帳號就進到共享的池子。每個你會用到的帳號都加一次。

3

一行指令切換

orrery use --claude personal 把該帳號設為 Claude 當前帳號。不用登出、不用重新認證 — 對話歷史在切換後完整保留。

4

接續上次的對話

切換帳號後，claude --resume 直接接續上一個 session。身分換了，延續性沒換 — 兩全其美。

5

進階：建立 sandbox

當某個客戶或專案需要完全獨立的 config，執行 orrery sandbox create client-a，再用 orrery enter client-a / orrery exit 進出。如果你只切帳號，這步可以完全跳過。

Phantom — 對話中切換

用 orrery run claude 啟動 Claude，會有一個 supervisor 守在旁邊。 在那個 Claude 裡，/orrery:phantom slash command 可以 不重啟對話直接換 account 或 sandbox — Claude 退出後 supervisor 帶著新設定與 --resume 把它叫回來，對話無感接續。

# 在以 `orrery run claude` 啟動的 Claude session 中
> /orrery:phantom personal
  # → 把 claude 帳號切到 'personal'

> /orrery:phantom codex work
  # → 切 codex 帳號

> /orrery:phantom sandbox client-a
  # → 進入 sandbox 'client-a'

Phantom 是 orrery run claude 的預設模式。 要關掉（單次執行、無 supervisor）：orrery run --non-phantom claude。 slash command 由 orrery mcp setup 自動安裝。

origin 基準狀態

你的預設 config — 還沒進任何 sandbox 時所在的地方

origin 就是你平常的設定：完全不用 Orrery 時你會用的那組帳號與工具 config。 首次執行 orrery setup 時，原本的工具設定（~/.claude/、 ~/.codex/、~/.gemini/）會被搬進 ~/.orrery/origin/， 原路徑改為 symlink — 資料本身完全沒動，只是改住在 Orrery 可以管理與同步的位置。 用 orrery use 切換帳號在 origin 就能做；只有當某個情境需要完全獨立的 config 時，才會用到 sandbox。

orrery exit 從任何 sandbox 切回 origin：Orrery 專屬的變數（如 CLAUDE_CONFIG_DIR）會被 unset，工具走自己原生的路徑 — 而這些路徑透過 symlink 指向 Orrery 管理的 config。從使用感受上像「離開 Orrery」，但實際上仍在 Orrery 管理的儲存之下。orrery enter origin 會被明確拒絕，並指向 exit：origin 是「沒有 sandbox」這個狀態，不是某個你會進去的 sandbox。

與一般 sandbox 不同，origin 無法被刪除或重新命名，且使用系統原生的 config 路徑。不過它支援 memory 與 session 設定：可以用 orrery sandbox memory isolate / share / storage 控制 memory 存放位置，也可以在 orrery setup 過程中選擇各工具的 session 是否隔離。執行 orrery sandbox info origin 查看完整狀態。

還原接管。 想讓工具完全脫離 Orrery（回到 setup 前的狀態），執行 uninstall — 一次把工具還原到原本 的位置並移除 shell 整合：

$ orrery uninstall
# 釋放全部 + 移除 shell 整合

應用情境

💼

工作 / 個人帳號

公司用 Claude Team plan，自己有 Pro 訂閱。兩個都加進池子，之後一行指令切換 — 完全不用登出。

$ orrery use --claude work
# → 公司 Team plan

$ orrery use --claude personal
# → 個人 Pro plan
💬

切換帳號不中斷對話

用工作帳號跟 Claude 對話到一半？切到個人帳號處理事情，再切回來 — session 還在原地等你。

$ orrery use --claude work
$ claude
# ... 工作中 ...

$ orrery use --claude personal
$ orrery use --claude work
$ claude --resume
# → 接續上次的對話
🏢

多客戶 / 多專案 SANDBOX

接案者或顧問同時服務多個客戶，每個都需要自己的憑證、設定與 memory。給每個客戶一個 sandbox 做完整隔離。

$ orrery sandbox create client-a
$ orrery sandbox create client-b
$ orrery enter client-a

AI 工具整合

Orrery 透過 MCP 整合 Claude Code、Codex CLI 和 Gemini CLI。跨帳號委派任務、工具間共享記憶、管理 session——全在一個地方。

1

設定一次就好

在專案目錄下執行。Orrery 會註冊為 MCP server 並安裝 slash commands。

$ orrery mcp setup
2

Slash commands

設定完成後，在 Claude Code 裡直接使用：

/orrery:delegate 委派任務給其他帳號
/orrery:delegate 用工作帳號 review 這個 PR 的安全問題
/orrery:sessions 列出 session 並用 index 接續
/orrery:sessions
接著： orrery resume 1
/orrery:phantom 中途切環境 — Claude 自動 --resume 接續對話
啟動時用： orrery run claude
在 Claude 內： /orrery:phantom work
3

或直接用自然語言

Claude 透過 MCP tools 認識你的所有環境。你可以這樣說：

→ 「用工作帳號 review 這個 PR」
→ 「我現在是哪個帳號？」
→ 「列出最近的 session，幫我接續上一個」

指令一覽

Account —每天都會用的那層
orrery add --<tool>新增帳號（--claude / --codex / --gemini）
orrery list [--tool]列出池子裡的帳號
orrery use --<tool> <name>切換某工具當前的帳號
orrery show顯示各工具當前的帳號
orrery remove從帳號池刪除帳號
Sandbox 狀態
orrery enter <name>在當前 shell 進入 sandbox
orrery exit離開目前的 sandbox 回到 origin
orrery which <tool>顯示工具設定目錄路徑
Sandbox CRUD
orrery sandbox create <name>建立 sandbox（wizard）
orrery sandbox delete [name]刪除 sandbox（不帶名稱會開多選）
orrery sandbox rename <old> <new>重新命名 sandbox
orrery sandbox list列出 sandbox
orrery sandbox info [name]顯示 sandbox 詳細
orrery sandbox current顯示目前啟用的 sandbox
Sandbox 設定
orrery tools add [-e <name>]加入工具（wizard）
orrery tools remove [-e <name>]移除工具
orrery sandbox set-env <K> <V>設定 sandbox 環境變數
orrery sandbox unset-env <K>移除 sandbox 環境變數
orrery sandbox memory storage <path>將記憶重導向至外部路徑
Session 管理
orrery sessions列出當前專案所有 session
orrery sessions --claude只看 Claude
orrery sessions --codex只看 Codex
orrery sessions --gemini只看 Gemini
Shell 整合
orrery setup安裝 shell 整合
orrery init輸出 shell function
orrery update自動更新（install script 或 Homebrew）
