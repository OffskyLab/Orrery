orrery v3.0 命令結構重整

Date: 2026-05-21 · Status: Draft · 把 account 提到 top level、env 改名為 sandbox 並降為 namespace

1. 為什麼要重構

orrery 第一版以 env 為核心抽象——所有指令都圍繞 env(orrery create / use / list ...)。 env 同時兼具「帳號隔離 + memory/sessions 沙箱 + env-var 容器」三個責任。

v2.8.0(PR #11)把帳號從 env 拆出來變成獨立 pool 之後,實際使用上看到: 大部分使用者只需要切換帳號、不在乎沙箱。env 已經變成「進階場景才會碰的東西」。 但目前命令面還是以 env 為主、account 在 nested namespace,這個結構繼續錯誤地暗示「env 是主、account 是附屬」。

2. 結構翻轉:env-centric → account-centric

v2.8 (現況)

env 是主、account 是附屬

Top-level — env 動詞
create use list delete info rename current deactivate memory sync env set/unset auth origin
account
add list show use remove
⇄
v3.0 (新)

account 是主、sandbox 是附屬

Top-level — account 動詞(預設 mental model)
add list show use remove
sandbox
create use list delete info rename current memory sync set-env unset-env

重點:list / use 這類動詞,在 v2.8 是 env 的、v3.0 變成 account 的—— 這是 hard break,但意義單一,不再有「list 看的是哪個」的雙重含意。 「env」改名為「sandbox」是為了避開既有的 env set/unset (環境變數操作)的語意衝突。

3. 設計目標

GOAL 1
入門 = account
第一次裝 orrery 的人,orrery add / orrery use / orrery list 就能用,不必先學什麼是 env。
GOAL 2
進階 = sandbox
需要 memory/sessions/env-var 隔離時才進 orrery sandbox *。
GOAL 3
動詞意義單一
list 永遠是 account、sandbox list 永遠是 sandbox。不會混淆。
GOAL 4
無歷史包袱
v3.0 一刀切、不留 alias。CHANGELOG 寫完整對照表,加錯誤訊息 hint 接住沒看的人。

4. 完整命令對應表

類別 v2.8 (現在) v3.0 (新) 備註
▎ env → sandbox namespace

sandboxorrery use <name>orrery sandbox use <name>含意改變
sandboxorrery listorrery sandbox list含意改變
sandboxorrery delete <name>orrery sandbox delete <name>
sandboxorrery info <name>orrery sandbox info <name>
sandboxorrery rename <old> <new>orrery sandbox rename <old> <new>
sandboxorrery currentorrery sandbox current
sandboxorrery memory ...orrery sandbox memory ...
sandboxorrery sync ...orrery sandbox sync ...
sandboxorrery export <name>orrery sandbox export <name>內部
sandboxorrery unexport <name>orrery sandbox unexport <name>內部
env-varorrery env set <k> <v>orrery sandbox set-env <k> <v>verb 一體化、避開 env namespace
env-varorrery env unset <k>orrery sandbox unset-env <k>
▎ account → top level
accountorrery account addorrery add
accountorrery account listorrery list含意翻轉

▎ 移除
removeorrery auth—功能被 show / list 取代
removeorrery origin—takeover 自動跑、release 走 uninstall
removeorrery deactivate—改用 orrery sandbox use origin
▎ 不動的正交動作
keeprun / delegate / sessions / resume同左
keepsetup / init / install / uninstall / update / check-update同左
keepmcp setup / mcp server / tools / third-party同左

orrery use <name> 預設 claude;同名跨工具時(work 在 claude/codex 都有)報錯並提示加 --<tool> 旗標。remove 同規則。

5. shell function 攔截變化

shell function 仍然需要攔截那些「Swift 進程改不到父 shell 環境」的指令(切 sandbox 要 export env vars / 設 ORRERY_ACTIVE_ENV),以及 TTY-attached 啟動 claude 的 case。

場景v2.8 攔截v3.0 攔截為何 shell 端要管
切 sandboxuse <name>sandbox use <name>需要 eval orrery-bin 算出的 env-var exports + 設 ORRERY_ACTIVE_ENV
切回 origindeactivate(移除)改用 orrery sandbox use origin
建立 sandboxcreate <name>sandbox create <name>創建完問「切過去?」需要 shell 端 read
add claude(TTY)account add (claude)add (claude)claude REPL 需要 foreground TTY
phantom clauderunrunphantom 迴圈;內容不變
切 accountaccount use不再攔截純 Keychain / symlink 操作,不需 shell magic

6. /orrery:phantom slash command

對話中觸發的 slash command 也跟 CLI 預設語意對齊——預設切 account(因為使用者已經在 claude session 裡),顯式 sandbox 關鍵字才切沙箱。

/orrery:phantom <name>
切 claude account(預設 — 因為 supervisor 本身是 claude session)
/orrery:phantom codex <name>
切 codex account(罕見,但允許)
/orrery:phantom gemini <name>
切 gemini account
/orrery:phantom sandbox <name>
切 sandbox(顯式 sandbox 關鍵字)
/orrery:phantom
無參數 → 列出可用的 account 與 sandbox,提示使用者

7. 內部 sentinel 欄位更名

phantom IPC sentinel (~/.orrery/.phantom-sentinel) 欄位:

TARGET_ENV='<sandbox-name>' TARGET_SANDBOX='<sandbox-name>' TARGET_ACCOUNT_TOOL='claude|codex|gemini' TARGET_ACCOUNT_NAME='<name>' SESSION_ID='<uuid>'

對應內部命令:PhantomTriggerCommand → PhantomSandboxTriggerCommand;_phantom-trigger → _phantom-trigger-sandbox(與 _phantom-trigger-account 對稱)。

⚠️ ORRERY_ACTIVE_ENV 環境變數保留原名——這是對使用者 script 的出口,改名會破壞既有 if [ "$ORRERY_ACTIVE_ENV" = ... ]。內部 Swift 命名以可讀性為主。

8. 相容性 hint(沒看 CHANGELOG 也能找到路)

v3.0 不留 alias,但加幾條友善錯誤訊息,接住舊使用者:

$ orrery use foo  (舊習慣;v3.0 是 account use,但沒有名為 foo 的 account、卻有同名 sandbox)
No account 'foo'. Did you mean: orrery sandbox use foo?
$ orrery create work  (已移除的 top-level)
Unknown subcommand 'create'. Did you mean: orrery sandbox create work?
$ orrery auth  (已刪除)
'orrery auth' was removed in v3.0. Use orrery show to see pinned accounts, or orrery list for the full pool.
